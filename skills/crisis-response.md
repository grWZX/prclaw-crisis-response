# 危机传播管理工具包 —— crisis_response

> **描述**：基于"事实-价值"模型和危机管理五路径的危机传播管理工具包。当组织面临突发事件、舆情危机或需要准备新闻发布会时使用，适用于企业、政府机构及公共事业部门。兼容 PRClaw Agent 架构。
>
> **版本**：v1.0 | **最后更新**：2026年6月24日 | **资源包**：`skills/crisis-response-toolkit/`

## 〇、目标

### 核心目标
将课件《第十讲：危机传播管理》中的"事实-价值"模型和五路径策略，转化为可执行的危机回应方案，使组织在黄金时间内完成从舆情输入到声明输出的全流程。

### 可衡量的成功标准

| 指标 | 目标值 | 衡量方式 |
| :--- | :--- | :--- |
| 响应时效 | 从输入到输出 < 5 分钟 | 时间戳记录 |
| 声明质量分 | ≥ 80 分（百分制） | `scripts/score_response.py` 自动计算 |
| 路径适配率 | 推荐路径与理论匹配度 ≥ 90% | 人工抽检 |
| 模板完整率 | 输出包含声明/Q&A/口径三类 | 结构化字段检查 |
| 三原则通过率 | 时间性、主体性、解释力 100% 覆盖 | 复核清单逐项确认 |

### 非目标（明确边界）
- 本 Skill 不负责原始舆情数据的采集（由铁路安全舆情智能体负责）
- 本 Skill 不代替法律合规审查（法律风险需人工复核）
- 本 Skill 不保证"转危为机"（课件明确指出这是错误想象，见第9页）
- 本 Skill 不负责危机后的长期声誉修复（那是恢复管理阶段的独立工作）

## 一、触发场景

| 场景类型 | 具体描述 | 典型触发词 |
| :--- | :--- | :--- |
| **突发事件响应** | 安全事故、产品质量问题、服务纠纷等 | "XX品牌出事了，帮我写声明" |
| **舆情危机处置** | 负面信息在社交媒体发酵 | "热搜第一，XX被骂上热搜" |
| **新闻发布会准备** | 需要召开新闻发布会 | "帮我准备发布会口径" |
| **声明质量复核** | 已有声明草稿需要评分优化 | "帮我看这份声明写得怎么样" |
| **内部口径统一** | 需要统一内部对外说法 | "给站段发一个内部口径" |

**对接智能体触发条件**：
- 负面情感指数 > 60%
- 单日声量增幅 > 500%
- 事件涉及关键词：安全事故/人员伤亡/设备故障/大面积晚点

## 二、理论框架（简要）

本 Skill 基于以下理论（详见 `crisis-response-toolkit/references/theory-framework.md`）：

1. **"事实-价值"模型**（课件§2）：危机传播 = 事实层（真相查证+利益互惠）+ 价值层（信任重建+意义分享）
2. **危机管理五路径**（课件§3）：告知 / 疏导 / 转换 / 顺应 / 引领
3. **新闻发布三原则**（课件§4）：时间性 / 主体性 / 解释力
4. **媒体应对五底线**（课件§4）：坦诚、说真话；对人、说人话；有效、说对话；修辞、说巧话；选择、说"糊"话

### 危机分级标准（Ⅰ–Ⅳ级）

| 等级 | 预警色 | 典型触发条件 | 首次表态 | 响应层级 |
| :--- | :--- | :--- | :--- | :--- |
| **Ⅳ级** | 🔴 红色 | 死亡≥30人（特别重大）；或死亡≥10人且引发全国性舆论 | 15分钟内 | 集团总部 + 政府联动 |
| **Ⅲ级** | 🟠 橙色 | 死亡3–9人；或受伤≥20人；或热搜级舆情 | 30分钟内 | 企业/集团公关总部 |
| **Ⅱ级** | 🟡 黄色 | 有限伤亡、区域性传播、上级限时通报 | 2小时内 | 事业部/局级公关 |
| **Ⅰ级** | 🔵 蓝色 | 轻微事件、无伤亡、低传播 | 4小时内 | 部门接口人 |

**强制升级规则**（伤亡量级下限，不可被关键词低估）：
- 死亡 ≥30 → 强制Ⅳ级红色
- 死亡 10–29 → 至少Ⅳ级红色
- 死亡 3–9 → 至少Ⅲ级橙色
- 重大伤亡事件策略路径：**告知 + 顺应 + 疏导**（态度话语优先，参考王勇平反面案例）

## 三、工作流程

```text
舆情事件输入（JSON）
    ↓
第一步：危机分级与诊断（Ⅰ/Ⅱ/Ⅲ/Ⅳ级）
    ↓
第二步：策略路径推荐（五路径组合）
    ↓
第三步：文案生成（初始声明/道歉信/Q&A/内部口径）
    ↓
第四步：三原则复核 + 质量评分（A/B/C/D级）
    ↓
输出最终方案（JSON）
```

详细流程说明见 `crisis-response-toolkit/references/spokesperson-checklist.md`。

## 四、输出模板

### 4.1 初始声明（30分钟内可用）

```text
【关于[事件名称]的初始声明】

[时间]，[地点]发生[事件简述]。我司/我局已第一时间启动应急预案，[相关领导]已赶赴现场。

目前，[已知事实简述]。详细情况正在进一步核实中。

我们将持续公布调查进展，回应公众关切。

[落款 + 时间]
```

### 4.2 正式道歉信

```text
【致歉信】

尊敬的[公众/相关方]：

近日，[事件简述]引发了社会关注，给您带来了困扰。对此，我们深表歉意。

经初步调查，[已查明的事实]。我们负有不可推卸的责任。

目前，我们已采取以下措施：
1. [具体措施一]
2. [具体措施二]
3. [具体措施三]

我们将继续深入调查，并在[时间节点]前公布详细结果。

[落款 + 时间]
```

### 4.3 媒体Q&A

| # | 问题 | 回答策略 |
| :--- | :--- | :--- |
| 1 | 发生了什么事情？ | 简述已确认事实；暂不能确认的说明原因 |
| 2 | 死伤/损失情况如何？ | 已确认数据如实说；尚不完全掌握的说明原因 |
| 3 | 谁对此负责？ | 明确责任主体 + 处理态度 |
| 4 | 事态是否得到了控制？ | 当前状态描述 + 后续安排 |
| 5 | 为什么会发生这样的事情？ | 已查明原因如实说；调查中的说明方向 |

### 4.4 内部口径

```text
【内部口径】关于[事件名称]的统一说法 ｜ 密级：内部

一、对外统一说法：[一句话概括]
二、统一表态口径：[标准表态]
三、具体事实口径：[已知事实]
四、暂时不可说事项：[统一回复"正在调查中"]
五、媒体应对注意事项：[不擅自接受采访]
```

## 五、质量标准

### 评分维度

| 维度 | 权重 | 评分标准 |
| :--- | :--- | :--- |
| **事实层** | 40% | 有具体事实（+20）、区分已知与调查中（+20） |
| **价值层** | 40% | 有道歉（+15）、有共情词汇（+15）、有承诺+时间（+10） |
| **路径适配** | 20% | 路径组合是否符合危机类型（+20） |

### 等级判定

| 等级 | 得分 | 处理建议 |
| :--- | :--- | :--- |
| A级 | ≥85 | 可立即发布 |
| B级 | 70-84 | 建议微调后发布 |
| C级 | 60-69 | 复核风险点后再发布 |
| D级 | <60 | 建议重新撰写 |

### 一票否决项

- 推卸责任（"个别员工""临时工"）
- 被动语态逃避主语
- 出现"无可奉告"
- 承诺整改但无时间节点

详见 `crisis-response-toolkit/references/spokesperson-checklist.md`。自动评分：`python skills/crisis-response-toolkit/scripts/score_response.py`

## 七、测试用例

### 测试1：高铁晚点事件
**输入**：G1234次列车因设备故障停运3小时，500名旅客滞留，抖音视频发酵。
**预期输出**：Ⅲ级响应，策略"告知+疏导"，得分75分（B级）。

### 测试2：明星失言风波
**输入**：艺人直播说"买不起XX是因为不努力"，上热搜第一。
**预期输出**：Ⅱ级响应，策略"顺应+疏导"，得分85分（A级）。

### 测试3：化工厂爆炸（事实未明）
**输入**：化工厂爆炸，有伤亡，原因调查中，省里要求18:00前通报。
**预期输出**：Ⅱ级响应，策略"告知+疏导"，得分82分（B级）。
