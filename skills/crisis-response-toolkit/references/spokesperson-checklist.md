# 发言人操作清单与禁忌速查 —— crisis-response-toolkit


## 一、新闻发布前检查清单

### 1.1 时间性检查

- [ ] 是否在事件发生后 **2小时内** 做出首次表态？
- [ ] 是否做到了 **"态度话语优先"** ——先道歉/致哀，再说理？
- [ ] 是否避免了 **"无可奉告""不便透露"** 等回避表述？

### 1.2 主体性检查

- [ ] 是否明确了 **责任主体**（谁在说话、代表谁）？
- [ ] 是否有 **标准故事版本**（统一口径）？
- [ ] 是否需要引入 **第三方意见领袖** 协助发声？

### 1.3 解释力检查

- [ ] 信息是否 **有生命力**（是否提供了公众关心的具体信息）？
- [ ] 是否使用了 **"是/故事/比喻/视觉"** 等修辞方式？
- [ ] 是否 **主动公开** 而非被迫公开？


## 二、发布中风险控制清单

### 2.1 事实层风险

| 风险 | 表现 | 对策 |
|---|---|---|
| 事实不清 | 在事实未明时声称"原因已查明" | 用"正在调查"替代 |
| 数据不可验证 | 说"有X人受伤"但无来源 | 标注来源或说明"待核实" |
| 信息矛盾 | 前后说法不一致 | 统一口径、专人发布 |

### 2.2 价值层风险

| 风险 | 表现 | 对策 |
|---|---|---|
| 推卸责任 | "个别员工""临时工" | 承担主体责任 |
| 冷漠 | "与我无关"的态度 | 表达关切和共情 |
| 粉饰危机 | "化危为机""转危为机" | 正视危机，不美化 |
| 诉讼威胁 | 以起诉威胁批评者 | 保持克制，接受监督 |

### 2.3 禁忌速查表

| 检查项 | 通过标准 |
|---|---|
| 是否出现"无可奉告" | 无 |
| 是否出现"个别人员""临时工" | 无 |
| 是否出现"深表遗憾"替代"对不起" | 无 |
| 是否出现"高度重视"但无具体动作 | 无 |
| 是否在事实未明时声称"原因已查明" | 无 |
| 是否出现"转危为机"等粉饰表述 | 无 |
| 是否出现"不便透露"等回避表述 | 无 |


## 三、媒体应对技巧

### 3.1 回答原则（课件第42页）

| 原则 | 含义 | 执行标准 |
|---|---|---|
| 坦诚，说真话 | 不撒谎、不隐瞒 | 所有事实陈述必须可验证 |
| 对人，说人话 | 避免官话套话 | 用"对不起"替代"深表遗憾" |
| 有效，说对话 | 针对性回应 | 回应公众最关心的议题 |
| 修辞，说巧话 | 适当的修辞技巧 | 使用"认同"策略 |
| 选择，说"糊"话 | 不确定时沟通不确定性 | "正在调查中，有结论将公布" |

### 3.2 现场应对技巧

| 情境 | 应对方式 |
|---|---|
| 记者提问超出已知范围 | "这个问题我们正在核实，有结论将第一时间公布" |
| 记者追问责任归属 | 承认总体责任，不推诿具体个人 |
| 记者质疑可信度 | 提供可验证的事实和数据 |
| 情绪激动时 | 先共情，再说理 |


## 四、策略路径对照速查

| 危机类型 | 推荐路径 | 禁忌 |
|---|---|---|
| 事实未明型 | 告知 + 疏导 | 不要过早下结论 |
| 责任明确型 | 转换 + 顺应 | 不要推卸责任 |
| 情绪高涨型 | 顺应 + 疏导 | 不要冷处理 |
| 内部混乱型 | 引领 + 告知 | 不要内外口径不一致 |
| 长期信任重建 | 引领 + 转换 | 不要只说空话 |

## 五、一票否决项速查

| 否决项 | 检测方法 |
|---|---|
| 推卸责任（"个别员工""临时工"） | 关键词匹配 |
| 被动语态逃避主语 | 语法模式检测 |
| 含具体数量但无数据来源 | 检测数字后是否标注来源 |
| 事实层为零、纯抒情 | 检测事实关键词个数 = 0 |
| 承诺整改但无时间节点 | 检测"将"后是否跟具体时间 |
| 出现"无可奉告" | 关键词匹配 |
